# Monkey Madness — iOS (SpriteKit-free CoreGraphics port)

A native iOS port of the HTML **Fart Back** prototype, using UIKit + `CADisplayLink`
+ Core Graphics (the closest 1:1 translation of the HTML `<canvas>` code, as the
design brief suggested).

## Status: gameplay-complete (Loud! theme) — builds & runs in the Simulator

The **full game** is ported and verified rendering in the iOS 26 Simulator:
- Swinging, farting monkeys on the branch; three monkey types (regular,
  Machine-Gun Marmoset, Boomer Baboon)
- Dodge + **Fart Back** (shoot a banana back + fart barrier)
- Banana types (yellow / brown mushy / black), slippery peels
- **Power-ups** (Gas Mask, Golden Banana, Nose Plug, Beano)
- **10-level campaign → King Kong-a-Toot boss (3 phases) → "$10" win**
- Juice (sticker pops, screen-flash, hit-stop), gas/lives/combo/score
- **Procedural fart audio** (AVAudioEngine), edge-to-edge background
- Start / level-complete / game-over / win screens, on-screen touch controls

**This is the SwiftUI/Xcode turning point.** What remains is naturally native-app
work: the other **3 swappable themes** (pure data — a `Theme` struct + swap), a
SwiftUI menu/settings shell, Game Center leaderboards, App Store assets, and the
TestFlight distribution below.

> Dev note: launch with the `AUTOPLAY` env var (`1` / `boss` / `win`) to jump
> straight into a state for screenshots.

## Build & run locally

```bash
cd ios
brew install xcodegen      # once, if you don't have it
xcodegen generate          # creates MonkeyMadness.xcodeproj from project.yml
open MonkeyMadness.xcodeproj
```
> The `.xcodeproj` is **generated** (not committed) — `project.yml` is the source
> of truth. Always run `xcodegen generate` after cloning or pulling.
Or from the command line (Simulator):
```bash
xcodebuild -project MonkeyMadness.xcodeproj -scheme MonkeyMadness \
  -sdk iphonesimulator -destination 'platform=iOS Simulator,name=iPhone 17' build
```

The app icon is generated by `swift scripts/makeicon.swift <out.png>`.

## Ship to TestFlight

Signing is configured: `project.yml` sets **`DEVELOPMENT_TEAM: L9SAXP2E2W`** with
automatic signing, and a clean **archive → export → signed `.ipa`** has been
verified locally. Bundle id: `com.daviddef.monkeymadness`.

**Before uploading:** in [App Store Connect](https://appstoreconnect.apple.com) →
Apps → **+ → New App**, create an app record with the **same bundle id**
(`com.daviddef.monkeymadness`). The upload is rejected without it.

### Option A — Xcode Organizer (easiest, no API key)
```bash
cd ios && xcodegen generate && open MonkeyMadness.xcodeproj
```
Set destination to **Any iOS Device (arm64)** → **Product ▸ Archive** → in the
Organizer, **Distribute App ▸ App Store Connect ▸ Upload**. Uses your signed-in
Apple ID.

### Option B — command line (uses your App Store Connect API key)
```bash
cd ios
xcodebuild archive -project MonkeyMadness.xcodeproj -scheme MonkeyMadness \
  -configuration Release -destination 'generic/platform=iOS' \
  -archivePath build/MonkeyMadness.xcarchive -allowProvisioningUpdates
xcodebuild -exportArchive -archivePath build/MonkeyMadness.xcarchive \
  -exportPath build/export -exportOptionsPlist ExportOptions.plist -allowProvisioningUpdates
# then upload (put AuthKey_9K9486HSDF.p8 in ~/.appstoreconnect/private_keys/):
xcrun altool --upload-app -f build/export/MonkeyMadness.ipa -t ios \
  --apiKey 9K9486HSDF --apiIssuer 69a6de8c-a266-47e3-e053-5b8c7c11a4d1
```

After either, the build appears in **TestFlight** (a few minutes to process); add
testers there. Bump `MARKETING_VERSION` / `CURRENT_PROJECT_VERSION` in
`project.yml` for each new build.
