# Monkey Madness — iOS (SpriteKit-free CoreGraphics port)

A native iOS port of the HTML **Fart Back** prototype, using UIKit + `CADisplayLink`
+ Core Graphics (the closest 1:1 translation of the HTML `<canvas>` code, as the
design brief suggested).

## Status: v0.1 — core loop

Ported and building/running in the iOS Simulator:
- Swinging, farting monkeys hanging from the branch
- Dodge + **Fart Back** (shoot a banana back + fart barrier)
- Banana types (yellow / brown mushy / black), gas meter, 4 lives, combo, score
- Start / Game Over screens, on-screen touch controls — **Loud!** theme

**Not yet ported** (queued): audio (fart synth via AVAudioEngine), the other 3
themes, power-ups, peels, levels + boss, and edge-to-edge background (currently
letterboxed to the 480×760 logical canvas).

## Build & run locally

```bash
cd ios
xcodegen generate          # regenerate MonkeyMadness.xcodeproj from project.yml
open MonkeyMadness.xcodeproj
```
Or from the command line (Simulator):
```bash
xcodebuild -project MonkeyMadness.xcodeproj -scheme MonkeyMadness \
  -sdk iphonesimulator -destination 'platform=iOS Simulator,name=iPhone 17' build
```

The app icon is generated by `swift scripts/makeicon.swift <out.png>`.

## Ship to TestFlight

TestFlight needs an **Apple Developer Program** membership ($99/yr) and your own
signing — these are account actions only you can do:

1. Open `MonkeyMadness.xcodeproj` in Xcode.
2. Select the **MonkeyMadness** target → **Signing & Capabilities** → check
   *Automatically manage signing* and pick your **Team**. (You may need to change
   the bundle id `com.daviddef.monkeymadness` to something unique to you.)
3. Set the run destination to **Any iOS Device (arm64)**.
4. **Product → Archive**.
5. In the Organizer: **Distribute App → App Store Connect → Upload**.
6. In [App Store Connect](https://appstoreconnect.apple.com): create the app
   record (same bundle id), then add the build to **TestFlight** and invite
   testers.

> The `project.yml` sets `CODE_SIGNING_ALLOWED: NO` so Simulator builds work
> without a team. Xcode's automatic signing overrides this for device/Archive
> builds once you select a Team.
